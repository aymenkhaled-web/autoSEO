Semantic Color Usage Rules:

Backgrounds: bg-background (page), bg-card (cards), bg-muted (subtle areas)
Text: text-foreground (primary), text-muted-foreground (secondary)
Borders: border-border
Accents: bg-primary text-primary-foreground (CTAs)
NEVER hardcode bg-white, bg-gray-900, text-black - always use semantic tokens
Status Colors (with dark variants):

Success: text-green-600 dark:text-green-400 bg-green-500/10
Warning: text-yellow-600 dark:text-yellow-400 bg-yellow-500/10
Error: text-red-600 dark:text-red-400 bg-red-500/10
Info: text-blue-600 dark:text-blue-400 bg-blue-500/10

4. UI COMPONENT LIBRARY
Use shadcn/ui exclusively - install all components upfront:

npx shadcn@latest add button card input label dialog dropdown-menu sheet tabs toast tooltip badge avatar select textarea checkbox switch radio-group separator scroll-area skeleton alert popover command form table progress

Toast notifications: Use sonner package
Icons: Use lucide-react ONLY - never mix icon libraries

5. SPACING & LAYOUT
Spacing scale (use these, no others):

Tight: gap-2, space-y-2 (8px)
Default: gap-4, space-y-4 (16px)
Comfortable: gap-6, space-y-6 (24px)
Generous: gap-8, space-y-8 (32px)
Sections: py-12 md:py-16 between major sections
Container widths:

Page wrapper: max-w-7xl mx-auto px-4 sm:px-6 lg:px-8
Forms/narrow: max-w-md mx-auto
Reading content: max-w-2xl mx-auto
Card padding: Always p-6 minimum, never less

19. ANIMATIONS & TRANSITIONS
Default transitions: transition-all duration-200
Page transitions: Fade in with animate-in fade-in duration-300
Hover effects:

Cards: subtle lift hover:shadow-md hover:-translate-y-0.5
Buttons: scale hover:scale-[1.02] active:scale-[0.98]
Use Framer Motion for complex animations only

20. RESPONSIVE BREAKPOINTS
Mobile first - design for 375px minimum
sm: 640px (large phones)
md: 768px (tablets) - sidebar appears
lg: 1024px (laptops) - full desktop layout
xl: 1280px (large screens)
Mobile-specific:

Stack everything vertically
Bottom nav instead of sidebar
Larger touch targets: min-h-[44px]
Hide secondary actions in dropdowns
Use sheets/drawers instead of side modals
21. ACCESSIBILITY (NON-NEGOTIABLE)
All interactive elements need focus rings: focus-visible:ring-2 focus-visible:ring-primary
Min contrast ratio 4.5:1 (WCAG AA)
All images need alt text
All inputs need <Label>
Use semantic HTML (<nav>, <main>, <article>)
ARIA labels on icon-only buttons
Keyboard navigation works everywhere
22. ICONS USAGE RULES
Source: lucide-react ONLY
Sizes: h-4 w-4 (inline), h-5 w-5 (default), h-6 w-6 (large)
Always paired with text-muted-foreground when decorative
Match text color when meaningful: text-primary, text-destructive
23. SHADOWS
Subtle: shadow-sm (cards default)
Default: shadow-md (hover state)
Elevated: shadow-lg (modals)
Glow effect: shadow-[0_0_30px_rgba(6,182,212,0.3)] (premium CTAs)
24. WHAT NEVER TO DO
❌ Don't hardcode colors (bg-gray-900, text-white)
❌ Don't use raw <button> or <input> - use shadcn
❌ Don't mix icon libraries
❌ Don't use rounded-md and rounded-lg randomly
❌ Don't skip dark mode
❌ Don't use spinners - use Skeleton
❌ Don't cramp content - use generous spacing
❌ Don't forget hover/focus/disabled states
❌ Don't use small touch targets on mobile
❌ Don't skip empty states
❌ Don't use default shadcn theme - customize HSL tokens
❌ Don't use Comic Sans, Arial, Times - only Inter
